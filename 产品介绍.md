# 设计需求平台 — 产品介绍

> **文档版本：V1**（当前对外说明与使用手册以此版为准；后续产品迭代将依次更新为 V2、V3……）

## 一句话说明

设计需求平台是看云内部使用的设计需求提交与管理工具，让需求方和制作人在一个地方完成从「提需求」到「交付」的全流程协作。

---

## 解决什么问题

| 以前 | 现在 |
|------|------|
| 需求散落在微信、邮件、口头沟通中 | 统一在平台上提交，所有人看到的信息一致 |
| 物料要什么尺寸、做几张说不清楚 | 每条物料单独填写类型、尺寸、数量，并附带对应文案 |
| 不知道需求进展到哪一步了 | 状态实时更新：待制作 → 制作中 → 待审核 → 已交付 |
| 谁改了什么、什么时候改的没记录 | 每次操作自动记录修改日志 |
| 想找之前的需求翻聊天记录 | 搜索、筛选、分页，历史需求随时查 |
| 同时写多个需求来回切换很麻烦 | 草稿箱支持保存多个草稿，随时继续编辑 |

## 核心功能

### 1. 账号与安全
- 公司邮箱（@kanyun.com）注册登录
- 邮箱验证：注册后需点击验证邮件才能登录，防止伪造账号
- 忘记密码：通过邮箱重置密码
- 两种角色：**普通用户** / **管理员**
- 管理员可在后台修改用户角色

### 2. 创建需求
- 基本信息：标题、提需人、部门（从企业微信复制）、需求类型（视觉设计/美术/动画）
- 期望交付日期与优先级（紧急/高/中/低），截止日期不能早于开始日期；版本提交节点（V1/V2…各对应提交日期与备注，自动编号）
- 需求背景与目标
- 需求物料 & 文案：每个物料独立填写类型、尺寸（可选或自由输入）、数量，以及对应的文案内容（模版或自由文本）
- 设计要求说明、参考图片上传、参考链接
- 补充说明
- 自动保存草稿，断网不丢

### 3. 草稿箱
- 支持同时保存多个草稿
- 侧边栏「草稿箱」入口，查看所有草稿
- 点击草稿继续编辑，提交后自动清除

### 4. 需求管理
- 首页统计面板：全部需求 / 待制作 / 制作中 / 待审核 / 已交付，点击卡片直接筛选
- 快速筛选：按类型、关键词搜索
- 我的需求：「全部」「我创建的」「我参与的」三个视角切换
- 逾期标红提醒
- 分页浏览，每页20条

### 5. 需求详情与协作
- 查看完整需求内容（物料与文案合并展示）
- 指派负责人（从注册用户中搜索选择）
- 添加协作人（协作人可编辑需求，支持自由输入姓名）
- 状态流转进度条 + 操作按钮：待制作 → 制作中 → 待审核 → 已交付
- 复制链接分享给同事
- 打印需求单
- 修改记录时间线

### 6. 权限控制
- 创建者和管理员可以删除需求
- 创建者、协作人、管理员可以编辑需求
- 创建者、指派的制作人、管理员可以变更状态
- 创建者和管理员可以指派制作人和协作人
