# 设计需求平台 — 产品介绍

## 一句话说明

设计需求平台是看云内部使用的设计需求提交与管理工具，让需求方和设计师在一个地方完成从「提需求」到「交付」的全流程协作。

---

## 解决什么问题

| 以前 | 现在 |
|------|------|
| 需求散落在微信、邮件、口头沟通中 | 统一在平台上提交，所有人看到的信息一致 |
| 物料要什么尺寸、做几张说不清楚 | 每条物料单独填写类型、尺寸、数量，一目了然 |
| 不知道需求进展到哪一步了 | 状态实时更新：待审核 → 设计中 → 已交付 → 已关闭 |
| 谁改了什么、什么时候改的没记录 | 每次操作自动记录修改日志 |
| 想找之前的需求翻聊天记录 | 搜索、筛选、分页，历史需求随时查 |

---

## 核心功能

### 1. 账号与权限
- 公司邮箱（@kanyun.com）注册登录
- 三种角色：**普通用户** / **设计师** / **管理员**
- 管理员可在后台修改用户角色

### 2. 创建需求
- 基本信息：标题、提需人、部门、需求类型（视觉设计/美术/动画）
- 需求背景与目标
- 期望交付日期与优先级（紧急/高/中/低）
- 需求物料：每一行填一种物料，选类型（海报/Banner/H5等）、选尺寸、填数量
- 需求文案：模版模式（主标题/副标题/正文）或自由文本
- 设计要求说明、参考图片上传、参考链接
- 补充说明
- 自动保存草稿，断网不丢

### 3. 需求管理
- 首页统计面板：全部需求 / 待审核 / 设计中 / 已交付
- 快速筛选：按类型、状态、关键词搜索
- 我的需求：「全部」「我创建的」「我参与的」三个视角切换
- 逾期标红提醒
- 分页浏览，每页20条

### 4. 需求详情与协作
- 查看完整需求内容
- 指派制作人（设计师）
- 添加协作人（协作人可编辑需求）
- 状态流转：待审核 → 设计中 → 已交付 → 已关闭（不能乱跳）
- 复制链接分享给同事
- 打印需求单
- 修改记录时间线

### 5. 权限控制
- 创建者和管理员可以删除需求
- 创建者、协作人、管理员可以编辑需求
- 创建者、设计师、管理员可以变更状态
- 创建者和管理员可以指派制作人和协作人
